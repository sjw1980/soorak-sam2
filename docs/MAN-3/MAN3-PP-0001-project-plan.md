## 1. 개발 대상 산출물

- `CLI Calculator` 소스코드 (Python)
- 단위 테스트 및 통합 테스트 (pytest)
- 정적 분석 결과 (Pylint 등)
- 사용자 매뉴얼 (README / Usage)
- 개발자 문서 (설계 요약, 빌드/테스트 절차)
- 빌드 아티팩트: 컨테이너 이미지 / 릴리스 번들
- 추적성 매핑표 (요구사항 ↔ 테스트 케이스)

## 2. 수명주기 모델/방법론

하이브리드(Incremental) — 짧은 반복(Iteration)과 주요 게이트(요구사항 확정, 구현 완료, 통합/QA, 릴리스)를 적용합니다. ASPICE 요구사항 준수를 위해 각 게이트에서 산출물·검증결과를 리뷰합니다.

## 3. 프로젝트 관리 관련 고객 요구사항

- 보고: 주간 상태 보고(간단한 텍스트 요약)
- 주요 게이트 리뷰 시 QA·테스트 결과 제출
- 최종 승인 권한: `유저` (Project Kickoff 문서 참조)

## 6. 추정 / 품질 기준

- 단위 테스트 커버리지: 분기 90% 이상 목표
- 정적 분석: Pylint 기준 주요 경고 해소
- 성능: 응답시간 100ms 이내(일반 연산)
- 메모리: RAM 64KB 이하(목표)
- 수용 기준: 요구사항 정합성 검증 및 테스트 케이스 100% 통과

## 7. 적용 프로세스 / 방법

- ASPICE 프로세스 적용: MAN.3, SWE-3(단위 설계·개발), SWE-4(단위 검증), SWE-5(통합)
- 코딩 규약: PEP 8 준수
- 형상관리: Git(브랜치 규칙: feature/, bugfix/, release/)
- CI/CD: GitHub Actions로 빌드·테스트 자동화

## 8. 대비책(리스크 및 대응)
