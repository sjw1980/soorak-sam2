### 1. 품질 관련 목표

- 단위 테스트 커버리지: 90% 이상(프로젝트 목표)
- 정적 분석: Pylint 주요 경고 해소(심각 오류 0)
- 결함: 릴리스 시점 `Critical` 결함 0, `Major` 결함 최소화
- 성능: 일반 연산 응답시간 < 100 ms
- 메모리: 런타임 메모리 목표 64 KB 이하
- 프로세스 준수: 게이트 체크리스트 항목 100% 완료

### 2. 품질 보장 활동 정의

- 리뷰
  - 요구사항 리뷰: 요구사항 베이스라인 시점, Requirement Owner 주관
  - 설계 리뷰: 상세설계 완료 시, Lead Architect 주관
  - 코드 리뷰(풀 리퀘스트 기준): 모든 변경은 PR + 최소 1명 승인
- 정적 분석
  - Pylint를 CI 파이프라인에서 실행, 주요 에러는 수정 후 머지
- 단위/통합 테스트
  - `pytest` 기반 단위테스트 자동화, coverage 리포트 수집
  - 통합 테스트는 통합 게이트에서 실행
- QA 점검(게이트 기준)
  - 각 게이트에서 체크리스트(templates/wp-13-19.md)를 사용하여 증거 제출
- 감사(선택 사항)
  - 필요 시 외부 또는 독립 QA의 감사 수행

### 3. 관련 산출물 참조

- 요구사항: `docs/SWE-1/*`, SRS(요구사항 리스트)
- 설계: `docs/SWE-3/*`, 상세설계
- 테스트: 단위/통합 테스트 케이스 및 결과 리포트
- 정적분석: Pylint 리포트
- 형상관리: Git 저장소, 릴리스 패키지
- 추적성: 요구사항↔테스트 매트릭스 (WP-13-22)

### 4. 규제/표준/고객 요구사항

- 적용 표준: ASPICE v3.1 (프로세스 요건 준수)
- 코딩 표준: PEP 8 권장, Pylint 규칙 적용
- 안전 규제(현재 해당 없음): ISO 26262 등은 본 프로젝트에서는 적용 대상이 아님(프로젝트 특성상)

### 5. 품질 기준 (Acceptance Criteria)

- 단위 테스트 커버리지 ≥ 90% (coverage.py/pytest-cov 측정)
- CI 통과(빌드/테스트/정적분석) 없이 릴리스 불가
- Pylint 심각 오류(fatal/error) 0건, major 경고는 릴리스 전 해소
- 모든 요구사항에 대해 대응되는 테스트 케이스 존재(추적성 매트릭스 상 확인)
- 통합 테스트 및 시스템 검증 통과

### 6. 감시/품질 점검 항목

- 측정 대상: 테스트 커버리지, 테스트 성공률, 열린 이슈 수(심각도별), Pylint 에러/경고 수, CI 빌드 상태
- 보고 주기: 주간 품질 리포트(간단 요약 + 주요 이슈)
- 책임자: QA/Test Lead가 지표 수집 및 주간 보고

### 7. 품질 보장 방법

- 자동화된 CI 파이프라인을 통한 정기적 검사(GitHub Actions)
- PR 기반 코드리뷰 및 Merge 규칙(승인자 1명 이상)
- 정적 분석 스크립트(Pylint) + coverage 리포트 자동수집
- 게이트(요구사항·설계·단위·통합·릴리스) 기반 검증 및 증거 제출

### 8. 시정 조치 임계값

- 단위 커버리지 < 90%: 시정 조치(추가 테스트 작성) 필요 — 릴리스 금지
- `Critical` 결함 발생: 즉시 대응, 릴리스 보류
- Pylint 심각 오류 발생: 수정 전 릴리스 불가
- 통합 테스트 실패율 > 10%: 원인 분석 후 리스크 완화 계획 수립

### 9. 품질 측정 방안

- 측정 도구: `pytest --cov`, `coverage.py`, `pylint`, GitHub Actions 빌드 로그
- 데이터 수집: CI 실행 결과 및 coverage/artifact 저장소에서 자동 수집
- 보고서: 주간 요약(정성적 + 정량적 지표)

### 10. 피드백 체계

- QA/Test Lead → Project Lead 주간 보고
- 게이트 결과 및 주요 품질 이슈는 게이트 회의에서 공유 및 결정
- 변경요청(CR) 발생 시 영향분석 후 우선순위 결정 및 이행

### 11. 객관성 보장 접근 방식

- 코드리뷰와 QA 담당자는 가능한 분리(개발자 ≠ QA 검토자)
- 중요 게이트에서 독립 리뷰 또는 외부 검토(필요 시)
- 체크리스트 및 증거 기반 검토(문서화된 기록 보관)
