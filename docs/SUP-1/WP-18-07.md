### 3. 코드 품질 기준

| 항목                     | 기준                | 비고                             |
| ------------------------ | ------------------- | -------------------------------- |
| 코드 복잡도 (Cyclomatic) | 함수당 평균 ≤ 10    | 복잡도 높은 함수는 리팩토링 대상 |
| 코드 커버리지            | 단위 커버리지 ≥ 90% | coverage.py/pytest-cov 측정      |
| Pylint 심각 오류         | 0 (fatal/error)     | CI에서 자동 검출                 |
| 코딩 표준 준수율         | 주요 규칙 위반 0건  | PEP 8 / Pylint 규칙 기반         |

### 4. 서비스 품질 기준

- 성능: 일반 연산 응답시간 < 100 ms(기본 시나리오)
- 메모리: 런타임 메모리 평균 목표 64 KB 이하
- 가용성(운영 환경): 프로젝트 규모상 단일 호스트 기준으로 서비스 가용성 목표 99% (해당 시 적용)

## 측정 및 보고

- 측정 주기: CI/빌드마다 자동 수집, 주간 리포트로 요약
- 도구: `pytest --cov`, `coverage.py`, `pylint`, GitHub Actions
- 보고서: 주간 품질 요약은 QA/Test Lead가 작성하여 Project Lead에게 제출
