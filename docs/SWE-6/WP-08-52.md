# SW 시험 계획서 (WP-08-52)

## 1. 목적

본 시험 계획서는 WP-19-00(적격성 전략)에 정의된 바에 따라 특정 릴리스 후보에 대해 실행할 시험의 범위, 방법, 자원, 일정, 진입/종료 기준 및 산출물을 상세화한다.

## 2. 적용 범위

- 대상 산출물: 통합된 소프트웨어 빌드 (RC)
- 포함 시험: 요구사항 기반 적격성 시험(QTC), 성능·자원 시험, 인터페이스 시험, 회귀 시험
- 제외: 단위 시험(별도 SWE-4 결과 참조)

## 3. 참조 문서

- WP-19-00 SW 적격성 테스트 전략
- SWE-1 요구사항 명세서
- SWE-2 아키텍처 문서
- 저장소 빌드/테스트 가이드 (CMake/Google Test)

## 4. 시험 항목

- 소프트웨어 아이템: `Calculator` 모듈, `InputParser`, AppController, 통합 실행 바이너리 등 (상세 항목은 빌드 목록에 따름)
- 외부 인터페이스: CLI I/O, 표준 입출력, 외부 데이터 파일 포맷

## 5. 시험 특성 및 우선순위

- 우선순위 1: 안전 관련 기능, 종료·에러 처리, 필수 기능
- 우선순위 2: 주요 비기능성(성능·자원), 경계 조건
- 우선순위 3: 예외 처리 및 부가기능

## 6. 시험 접근법

- 모든 QTC는 SWE-1 요구사항 ID와 매핑됨
- 테스트 설계: 요구사항 → 시험케이스 도출 → 시나리오화 → 자동화(우선)
- 수행 방식: 자동화된 케이스 우선 실행, 수동/시나리오성 케이스는 실행 로그·스크린샷 첨부

## 7. 시험 환경 및 도구

- 빌드: CMake 3.16+, Debug 빌드(커버리지 수집 시) 권장
- 컴파일러: gcc/g++ 또는 프로젝트 승인된 컴파일러
- 테스트 프레임워크: Google Test (FetchContent 사용 권장)
- 실행: CTest
- 커버리지: gcov/lcov 또는 gcovr

### 7.1 구성/실행 명령 예시

```
# 구성
cmake -B build -DCMAKE_BUILD_TYPE=Debug

# 빌드
cmake --build build

# 테스트 실행
ctest --test-dir build -V

# (옵션) 커버리지
lcov --capture --directory build --output-file coverage.info
genhtml coverage.info --output-directory coverage_report
```

## 8. 시험 케이스 계획

- 시험케이스는 고유 ID(QTC-xxxx)로 관리
- 각 케이스는 다음 필드를 포함: 목적, 전제조건, 입력 데이터, 절차, 기대 결과, 요구사항 ID
- 추적성 매트릭스는 `docs/SWE-6/WP-08-52-TRACE.md`에 저장

## 9. 진입/종료 기준

### 9.1 진입 기준

- 통합 빌드가 성공적으로 생성되고 제공됨
- 단위·통합 테스트의 심각 결함이 열려있지 않음(또는 대체 승인)
- 테스트 환경 및 데이터 준비 완료

### 9.2 종료 기준

- 모든 우선순위 1 케이스 PASS
- 우선순위 2 케이스는 PASS 또는 승인된 이관
- 모든 발견된 결함은 SUP-9 프로세스에 따라 관리되고 상태가 정의됨
- 시험 보고서(적격성 시험 보고서) 작성 및 승인

## 10. 자원 및 책임

- 테스트 리드: 시험 일정·진행 관리, 결과 승인
- QA 엔지니어: 시험케이스 설계·실행·결과 기록
- 개발자: 결함 수정·재시험
- 구성관리: 빌드·아티팩트 관리
- 필요 인력: 테스트 리드 1명, QA 엔지니어 2명(규모에 따라 조정)

## 11. 결함 관리

- 결함은 SUP-9 문제관리 프로세스에 등록
- 심각도 기준: P1(치명) / P2(주요) / P3(경미)
- 결함 상태 및 우선순위에 따라 재시험 일정 조정

## 12. 시험 일정(예시)

- 시험 케이스 설계: 2026-04-20 ~ 2026-04-24
- 시험 환경 준비: 2026-04-22 ~ 2026-04-24
- 시험 실행(1차): 2026-04-25 ~ 2026-04-28
- 결함 수정 및 재시험: 2026-04-29 ~ 2026-05-03
- 최종 보고 및 승인: 2026-05-04

(일정은 프로젝트 계획에 따라 조정)
