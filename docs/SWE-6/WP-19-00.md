# SW 적격성 테스트 전략 (WP-19-00)

## 1. 목적

이 문서는 ASPICE SWE-6 (SW 적격성) 요구사항에 따라 소프트웨어 적격성(acceptance) 시험의 고수준 전략을 정의한다. 본 전략은 제품 인도 전 최종 통합 SW가 기능적·비기능적 요구사항을 충족함을 검증하기 위한 원칙, 범위, 시험 수준, 도구 및 진입/종료 기준을 제시한다.

## 2. 적용 범위

- 대상: 릴리스 후보(Release Candidate)로 빌드된 통합 소프트웨어 산출물
- 포함 항목: 기능 적격성(요구사항 기반), 성능·자원·신뢰성 적격성, 인터페이스(외부 시스템 포함) 검증
- 제외 항목: 단위 테스트(해당 내용은 SWE-4), 상세 통합 테스트(일부 SWE-5 항목은 상호 연계)

## 3. 참조 문서

- SWE-1 요구사항 명세서
- SWE-2 아키텍처 설계 문서
- SWE-4 단위 검증 결과 및 SWE-5 통합 시험 결과
- WP-08-52 (시험 계획서)
- 저장소 내 빌드/테스트 가이드 (`gcc-test-build` 스킬 문서)

## 4. 전제 조건

- 통합 빌드가 성공적으로 생성되어야 함 (빌드 산출물: `build/` 디렉토리)
- 관련 단위 테스트 및 통합 테스트가 완료되어 주요 결함이 해결되어야 함
- 적격성 테스트용 하드웨어/시뮬레이션 환경(필요한 경우) 준비 완료

## 5. 전략 요약

- 목표: 시스템 수준에서 SW 요구사항이 충족되는지 확인하여 고객 인수/출시에 적합함을 판단
- 접근법: 요구사항 기반 테스트(요구사항 ↔ 시험케이스 1:1 매핑), 시나리오 기반 검증, 비기능성(성능·자원·신뢰성) 시험 포함
- 우선순위: 안전·제어·오류 처리 관련 요구사항 우선 검증

## 6. 시험 수준 및 유형

- 시험 수준: 통합(Integrated) → 적격성(Acceptance)
- 시험 유형:
  - 기능 적격성 테스트 (요구사항 기반)
  - 인터페이스/통신 시험
  - 성능·자원(메모리/CPU) 시험
  - 신뢰성(내구성, 장시간 실행) 시험
  - 회귀(Regression) 시험 — 릴리스 후보에 대한 최종 회귀

## 7. 설계·기법

- 요구사항 기반 명세(명세기반 테스트)
- 시나리오 및 사용사례 기반 검증
- 경계값 분석, 등가분할, 오류 주입(fault injection) 등
- 자동화 우선: 가능한 경우 테스트 자동화(CTest/Google Test + CI) 사용

## 8. 시험 환경 및 도구

- 운영체제: 검증 대상 플랫폼(예: Linux, Windows) 명시
- 빌드 도구: CMake 3.16+ (프로젝트 정책에 따름)
- 컴파일러: gcc/g++ 권장 (환경에 따라 MSVC 등 허용)
- 테스트 프레임워크: Google Test (권장 v1.14.0 via FetchContent)
- 실행/자동화: CTest, `ctest --test-dir build -V` 사용 권장
- 커버리지/분석: gcov/lcov 또는 `gcovr` (Debug 빌드에서 수집)

참고 명령 예시:

```
# 구성
cmake -B build -DCMAKE_BUILD_TYPE=Debug

# 빌드
cmake --build build

# 테스트
ctest --test-dir build -V

# 커버리지 (예시)
lcov --capture --directory build --output-file coverage.info
genhtml coverage.info --output-directory coverage_report
```

## 9. 커버리지 및 적격성 기준

- 적격성 판단의 1차 기준: 요구사항 명세서에 할당된 QTC(적격성 시험케이스) 모두 `PASS`
- 비기능성: 성능·자원·신뢰성 지표는 요구사항 또는 프로젝트 합의 기준을 만족해야 함
- 커버리지: 적격성 수준에서는 요구사항 커버리지 100% 달성(요구사항 ↔ 시험케이스 매핑으로 확인)

## 10. 진입/종료 기준

- 진입 기준:
  - 통합 빌드 성공
  - 주요 단위·통합 시험 통과(심각 결함 없음)
  - 테스트 환경 및 데이터 준비 완료
- 종료 기준:
  - 모든 필수 QTC PASS
  - 모든 우선순위 1·2 결함 해결 또는 승인된 리스크로 이관
  - 적격성 시험 보고서 승인

## 11. 역할 및 책임

- 테스트 리드: 시험 계획 집행·일정 관리·결과 최종 검토
- QA 엔지니어: 시험케이스 설계·테스트 실행·증적 수집
- 개발자: 결함 수정·재시험 지원
- 구성관리: 빌드·릴리스 아티팩트 관리

## 12. 리스크 및 완화책

- 빌드 불안정: 빌드 단계 자동화 및 빌드 기준 문서화
- 테스트 환경 불일치: 환경 프로비저닝 스크립트 및 가상화/컨테이너화 검토
- 테스트 데이터 부족: 시나리오별 데이터 생성 계획 수립

## 13. 추적성

- 모든 QTC는 SWE-1 요구사항 ID와 1:1 매핑되어야 하며, 추적성 매트릭스는 `docs/SWE-6/WP-08-52-TRACE.md`(또는 별도 엑셀/CSV)로 관리한다.
